## 🙇🏻‍♀️ Introduction

### 문제 인식 및 문제 정의
#### (1) 문제 인식
  대학가 상권은 학생 유동 인구에 크게 의존하는 구조를 가지고 있기에 학기와 방학에 따라 유동 인구와 매출의 변동 폭이 크고,
이로 인해 캠퍼스 주변 상권에서는 폐업 점포 수가 지속적으로 증가하고 있다. 실제로 중앙대학교 인근 상권에서도 분기별 폐업 점포 수가 꾸준히 증가하고 있으며, 이는 단순한 개별 점포의 문제가 아닌 지역 상권 전반의 구조적 문제로 이어지고 있다.

<img width="1231" height="653" alt="image" src="https://github.com/user-attachments/assets/b73ac9b8-ae51-453d-a74a-74ac1ce4e4e6" />

#### (2) 문제 정의
학생단체와 상점은 제휴의 필요성은 인식하고 있지만, 서로에게 적합한 제휴 조건을 설계하고 판단할 기준이 부족하기에 제휴 제안부터 조건 설계, 체결까지를 데이터 기반으로 연결해주는 통합적인 제휴 시스템이 필요하다.
<img width="1235" height="650" alt="image" src="https://github.com/user-attachments/assets/9329d042-647a-4cf3-bfc6-37dfeeec7d8f" />


## ✨ Key Features

- **AI 제휴 제안서 생성**  
  학생단체와 상점의 조건을 분석하여 제휴 제안서를 자동으로 생성한다.

- **제휴 대상 탐색**  
  제휴 가능한 상점과 학생단체를 한눈에 탐색할 수 있다.

- **양방향 제휴 제안**  
  학생단체와 상점이 서로 제휴를 제안하고 협의할 수 있다.
